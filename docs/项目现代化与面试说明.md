# nest-admin 现代化与面试说明

## 项目定位

nest-admin 是一个基于 NestJS 的后台权限管理服务，覆盖后台系统常见的用户、角色、菜单、部门、日志、任务、在线用户、网盘和 WebSocket 通知能力。它适合作为 Node.js 后端、NestJS 工程化和 RBAC 权限模型的作品集项目。

## 当前技术栈

- Runtime: Node.js 22+
- Framework: NestJS 11
- Language: TypeScript 5
- HTTP Adapter: Fastify
- ORM: TypeORM 0.3
- Database: MySQL 8.4
- Cache and session state: Redis
- Auth: JWT + Redis token state + password version
- Queue and scheduled jobs: Bull
- Docs: Swagger / OpenAPI
- Logs: Winston + daily rotate files
- Delivery: Docker, Docker Compose, GitHub Actions

## 核心业务能力

- 登录验证码、JWT 签发、Redis 登录态管理。
- RBAC 权限控制，菜单权限粒度可映射到接口路径。
- 用户、角色、部门、菜单、参数配置等后台基础模块。
- 操作日志、登录日志、在线用户踢下线。
- Bull 任务调度和任务执行日志。
- WebSocket 在线状态和菜单更新通知。
- 七牛云网盘文件管理能力。

## 本次现代化升级

1. 将运行时目标提升到 Node.js 22，并升级 NestJS、TypeScript、Fastify、TypeORM 等核心依赖。
2. 将密码存储从 MD5 + salt 升级到 Argon2。
3. 保留旧 MD5 用户兼容逻辑：旧用户登录成功后自动重写为 Argon2 哈希。
4. 生产环境默认关闭 TypeORM `synchronize`，改用 migration 流程管理 schema。
5. 增加 GitHub Actions CI，覆盖 lint、test、typecheck、build 和 Docker build。
6. Docker 基础镜像固定到 Node 22，MySQL 镜像固定到 8.4，减少 `latest` 带来的不确定性。

## 面试可讲点

- 为什么选择 NestJS：模块化、依赖注入、守卫、拦截器、过滤器和 Swagger 生态适合后台管理系统。
- 权限设计：JWT 负责身份，Redis 保存登录态和权限快照，Guard 统一做接口级权限校验。
- 密码升级方案：不强制所有旧用户重置密码，而是采用兼容校验 + 登录后 rehash，降低迁移风险。
- 数据库发布策略：开发环境可以同步表结构，生产环境关闭 `synchronize`，使用 migration 可审计地变更 schema。
- 工程化：CI 将 lint、test、typecheck、build、Docker build 放入质量门禁。
- 部署：Docker Compose 本地编排 MySQL、Redis、前端和后端服务，生产可替换为独立数据库和缓存服务。
